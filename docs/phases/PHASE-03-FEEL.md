# Phase 3 — Feel

**Status:** complete  
**Depends on:** [PHASE-02-CORE-LOOP.md](./PHASE-02-CORE-LOOP.md)

## Goals (MASTER SPEC — Phase 3)

- **Howler.js** — BGM loop + SFX bus driven by `settingsStore` (master / BGM / SFX / mute).
- **Gameplay → audio** — tick emits `GameFeelEvent[]`; `playFeelEvents` maps to SFX. Store handles wave start, build, victory, game over.
- **GSAP** — full-screen **Wave** callout when a wave runtime begins (`WaveBanner.tsx`).
- **Motion** — procedural **bob** on enemies (`useFrame`); archer **Y rotation** toward current target (`TowerInst.facingY`).
- **Pooling scaffold** — `objectPool.ts` ring buffer (simulation still uses immutable arrays).
- **Tests** — `npm test` runs Vitest on `pathfinding.test.ts`.
- **Placeholder audio** — generated by `node scripts/generate-placeholder-audio.mjs` into `public/audio/*.wav` (replace with real assets anytime).

## Performance notes

- **Stats** (Phase 1) remains the primary dev FPS overlay; Phase 3 did not add a production HUD FPS meter.
- **Simulation** still allocates new arrays per tick; swapping to pooled projectile buffers is deferred to a later optimisation pass.
- BGM + multiple Howl instances increase bundle I/O; for shipping, prefer **one** SFX sprite sheet + compressed **OGG/WebM** secondary sources in Howler `src` arrays.
